#### Маска А на 16
```
Класс A: 255.0.0.0
log2(16) = 4 (1111)

11110000 = 240

Ответ: 255.240.0.0

```


#### Маска B на 8 подсетей
```
Класс В: 255.255.0.0
log2(8) = 3 (111)

11100000 = 224

Ответ: 255.255.224.0

```


#### Маска С на 4
```
Класс C: 255.255.255.0
log2(4) = 2 (11)

11000000 = 192

Ответ: 255.255.255.192
```


#### Класс для IP 250.98.1.0
```
dec: 250.98.1.0
bin: 11111010.01100010.00000001.00000000

Первые 4 бита адереса равны 1111, что означает его принадлежность сети класса Е.

Ответ: E

Почему не F, ответ: https://www.cisco.com/c/ru_ru/support/docs/ip/routing-information-protocol-rip/13788-3.html

```

#### К какому классу относится сеть 128.10.23.45
```
dec : 128.10.23.45
bin*: 10000000.10.23.45

Первые 2 бита адерса равны 10, что означает его принадлежность сети класса А.

Ответ: А

```


#### К какой подсети относится такой-то IP
```
Пусть дан ip-адрес:
dec: 128.10.23.45
bin: 10000000.00001010.00010111.00101101

class B mask:
bin: 255.255.0.0
bin: 11111111.11111111.00000000.00000000

Адрес (в) подсети:
x - биты номера сети
bin: xxxxxxxx.xxxxxxxx.00010111.00101101


Для того, чтобы знать к какой подсети относится данный адрес нужно
зать либо сколько всего подсетей, либо какой это узел по счету


1. Если известно, что всего подсетей 16, тогда

log2(16) = 4 - кол-во первых бит адреса определяющих номер подсети
y - биты номера подсети

10000000.00001010.00010111.00101101
xxxxxxxx.xxxxxxxx.yyyy0111.00101101
yyyyy = 00001

00001 = 1, следовательно это превая подсеть


2. Если известно мин. кол-во узлов в сети 3885

log2(3885) = 11.9 =(округляем в большую)= 12
Забиваем последние 12 бит адреса под номер узла


10000000.00001010.00010111.00101101
xxxxxxxx.00000000.0000zzzz.zzzzzzzz

оставшиеся биты указывают номер подсети

00001 = 1, следовательно это превая подсеть


ПРИ ИНЫХ УСЛОВИЯХ, ОТВЕТЫ БУДУТ ИНЫМИ

```
